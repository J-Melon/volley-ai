---
name: Only create issues when cycle needs more points
description: Don't proactively create new issues; only create them if the next cycle doesn't have enough points queued
type: feedback
---

Only create new Linear issues if the upcoming cycle doesn't already have enough points to fill it. Don't create issues just because gaps are identified in planning or design review sessions.

**Why:** Issue creation should be demand-driven by cycle capacity, not by completeness instinct. Identifying a gap is useful; immediately filing an issue for it is premature if the cycle is already full.

**How to apply:** When gaps are found during planning or design review, note them but hold off on creating issues. Only proceed to create issues if the user asks, or if there's an explicit need to fill cycle capacity.

Volley is in development. We produce working specs, drafts, fiction, design. Settled-truth vocabulary about a finished work is generated by players engaging with the published game, not by developers shipping toward it. That vocabulary belongs to the player side and has nothing to do with us.

**Banned from memory bodies, memory file names, `ai/skills/**`, `CLAUDE.md`, agent dispatch docs, and design / narrative / character / concept docs:**

- The five-letter player word for settled fiction in a published work (alphabetises between "candy" and "cant", ends in "-non", what fans cite as officially established). Replace with "fiction", "design truth", "settled", "established", "the work", or name the doc directly.
- "final" carries the same shape. Replace with "current", "working", "latest". The exception is sequence-ordinal ("the final memory yields a key", "the final milestone"): last-in-sequence survives.

**Why:** sharpened 2026-05-24 on the SH-423 outline-rework. The player-side word names a thing audiences derive from a finished game. We don't have that thing and never will, because it isn't generated at the maker side. Using the word at the development surface plants a settled-truth weight that doesn't belong there and licences speculation to read as settled.

**How to apply:** grep for the two before saving any memory or editing any process doc. If either appears, replace. The oblique reference above is the one allowed mention of the player word, in order to ban it without using it.

Write a memory only after the work it describes is done and confirmed, not mid-flight and not on
a hunch about how something will turn out. A memory captured before the work is a guess, and
guesses get reversed; recording one early means writing it, then rewriting it, then maybe deleting
it, all in the same session (the churn is the tell).

Why: a mid-task memory enshrines my own coherence before any external signal has confirmed it,
which is exactly the trap in [[feedback_self_judgment_is_coherence_not_accuracy]]. Doing the work,
and seeing it hold, IS that external signal. So the work has to come first.

This session (2026-06-03) is the corpus: the "don't bug me" rule was written, then flipped when
Josh said the opposite; the "unlink dead PRs" rule was written, then undone when he said links
accumulate; a `spec`-vs-`spike` label memory was written, then deleted when the call reversed.
Each was recorded before the decision settled.

How to apply: do the task. Let the correction or confirmation land. THEN, at a natural stopping
point, write the memory from what actually proved true, not from the plan. Josh, 2026-06-03:
"always write memories AFTER doing the work." Pairs with committing it promptly once written
([[feedback_commit_memory_promptly]]).

The flip side, same axis: **a retro thought IS the trigger to write.** When I notice myself producing
a "here is the pattern under what just happened" reflection (the named takeaway after a correction
or a chain of them), that reflection is the cue to write or sharpen a memory, not a thing to admire
and move past. Josh, 2026-06-03: "whenever you have thoughts like that consider a memory." The
failure mode this session was the opposite of writing-too-early: having the validated insight and
SKIPPING it, talking myself out with "a parent rule already covers the territory." The test is not
"does a rule exist" but "does this turn sharpen one (a new mechanism, tell, or trigger)"; if yes,
write it. So: after the work, when the retro thought lands, that is the moment, not later.

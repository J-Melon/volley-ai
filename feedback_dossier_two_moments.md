The Lair guide currently names Dossier as pre-cycle work only ("Before the cycle opens, Josh and Gru assemble the Dossier..."). That's half. The other half is mid-cycle: when the current cycle starts running low on Ready issues and the team is about to starve for work, Gru dandoris the Dossier again to top the cycle up from Vault.

Two dandori moments for the Dossier:

1. **Pre-cycle (flip).** Before a cycle opens, assemble the Ready set for the upcoming two weeks. Estimated, labelled, design-linked.
2. **Mid-cycle (top-up).** When Ready issues in the current cycle are about to run out (agents finishing faster than issues flow in), Gru dandoris the Dossier again to promote Vault issues into Ready and into the cycle.

**Why:** Reinforced 2026-04-24 after Kyle Patrol's parallel Challenges finished faster than I expected. Josh named the second moment: "there are two times we dandori the Dossier, before next cycle or when we are running out of issues for this one." The Dossier is a recurring ritual, not a one-shot pre-cycle event.

**How to apply:**
- Watch cycle Ready-issue depth actively. When unstarted Ready count drops to roughly one dispatch-unit's worth, flag to Josh that a mid-cycle Dossier top-up is due.
- The mid-cycle top-up pulls from the same project(s) the cycle is already working, unless Josh redirects.
- The pre-cycle Dossier is larger in scope (full cycle shape); the mid-cycle Dossier is narrower (a few issues to keep the queue running).
- Both moments use the same verb: Gru dandoris the Dossier. Same ritual, two triggers.

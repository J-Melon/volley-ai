When Josh references "CI" or "under CI" in a planning context, he means the **continuous-improvement principle** - the standing license to land small, in-flight scope improvements without spinning up a dedicated project or waiting for a planned milestone. He does **not** mean a Linear project named CI (none exists) and does **not** mean GitHub CI/CD.

**How to apply:** When Josh frames a follow-up as "under CI", treat it as authorisation to file a small ticket and dispatch a minion now, rather than dandori-ing it into a project. Don't propose project homes; just scope the work, file the issue, dispatch.

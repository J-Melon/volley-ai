---
name: Tickets pulled cross-project stay in the new project
description: When a mission pulls an issue out of one Linear project into another, it stays there permanently
type: feedback
originSessionId: d15b1172-9e53-401f-b338-5c126b669606

When bundling an issue from another Linear project into a mission, the issue moves to the host project and stays there. Don't plan to "return it after the mission".

**Why:** Project membership reflects where the work actually lived and shipped. Bouncing tickets back after the fact lies about scope, makes velocity data noisy, and forces a second admin step nobody benefits from. If the pull was wrong, that's a sign the project boundary itself is wrong.

**How to apply:** When proposing a mission that crosses project boundaries, frame the move as permanent. Pick the host project deliberately. If the pulled ticket really belongs back in its old home, that means it shouldn't have been pulled at all.
